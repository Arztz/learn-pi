# Extensions intro

Extensions are TypeScript modules that add new capabilities to pi — custom tools, commands, event handlers, and UI components.

## What Extensions Can Do

| Capability | Description | Example |
|------------|-------------|---------|
| **Custom Tools** | Add LLM-callable tools | `deploy`, `query-db` |
| **Commands** | Slash commands like `/deploy` | `/hello`, `/stats` |
| **Event Handlers** | React to tool calls, messages | Auto-commit on each turn |
| **UI Components** | Custom overlays, status lines | Doom game overlay |
| **Permission Gates** | Confirm destructive actions | Block `rm -rf /` |

## Extension Locations

| Location | Scope |
|----------|-------|
| `~/.pi/agent/extensions/` | Global (all projects) |
| `.pi/extensions/` | Project-local |

Drop a `.ts` file in one of these directories and pi auto-loads it on startup.

## Anatomy of an Extension

```typescript
import type { ExtensionAPI } from "@mariozechner/pi-coding-agent";

export default function (pi: ExtensionAPI) {
  // Register a tool
  pi.registerTool({ name: "my-tool", ... });

  // Register a command
  pi.registerCommand("hello", { description: "...", handler: ... });

  // Listen to events
  pi.on("tool_call", async (event, ctx) => {
    console.log(`Tool ${event.toolName} was called`);
  });
}
```

## Loading Extensions

Extensions load in this order:

1. Global extensions (`~/.pi/agent/extensions/`)
2. Project extensions (`.pi/extensions/`)
3. Settings.json extension paths
4. Command-line `--extension` flags

Use `/reload` to hot-reload extensions without restarting pi.
