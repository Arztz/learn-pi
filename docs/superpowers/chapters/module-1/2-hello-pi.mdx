# Hello, pi!

Time to send your first prompt. This chapter teaches you what pi's responses look like and how tool calls work.

## Your First Prompt

Run pi and type:

```
Hello, what can you do?
```

pi will respond with a friendly message explaining its capabilities. Since this is just a conversation, no tools are needed yet.

## Tool Calls Explained

When pi needs to actually DO something (read a file, run a command, write code), it uses **tool calls**.

Here's what happens when you ask pi to list files:

```
> List files in the current directory

◐ Thinking...
  The user wants to see what files are in the current directory.
  I should use the `bash` tool to run `ls`.

ℹ bash { command: "ls" }

✓ Tool: bash
   src/
   package.json
   README.md
   tsconfig.json

That's the contents of your current directory!
```

### Anatomy of a Tool Call

| Part | Example | Meaning |
|------|---------|---------|
| **Thinking block** | `◐ Thinking...` | Shows pi's reasoning (collapsible) |
| **Tool invocation** | `ℹ bash { command: "ls" }` | The tool being called |
| **Result** | `✓ Tool: bash` + output | The tool's execution result |

## Try It Yourself

In your terminal:

```bash
pi
```

Then type these prompts one at a time:

1. `"Hello!"` — Just a greeting, no tools
2. `"What files are in this directory?"` — Uses bash tool
3. `"Read the README.md file"` — Uses read tool

Watch how pi decides whether to use a tool!

## The Four Built-in Tools

| Tool | Purpose | When to Use |
|------|---------|-------------|
| **read** | Read file contents | Viewing code, configs, docs |
| **write** | Create new files | New files, complete rewrites |
| **edit** | Modify existing files | Targeted changes, patches |
| **bash** | Run shell commands | Git, npm, file ops, compilation |
