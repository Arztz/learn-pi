# Sessions

pi saves your conversations as **session files** — JSONL files with a tree structure that lets you navigate history.

## How Sessions Work

Every time you start pi, it creates or continues a session:

```bash
pi                    # Continue most recent session
pi -c                 # Same as above (continue)
pi -r                 # Resume - browse and pick a session
pi --session <id>     # Open specific session
```

Sessions are stored in `~/.pi/agent/sessions/` organized by project directory.

## Session File Format

Sessions are **JSONL** (JSON Lines) — one JSON object per line:

```json
{"type":"session","version":3,"id":"abc123","cwd":"/project"}
{"type":"message","id":"001","parentId":null,...}
{"type":"message","id":"002","parentId":"001",...}
{"type":"message","id":"003","parentId":"002",...}
```

### Entry Types

| Type | Purpose |
|------|---------|
| `session` | Header with version, ID, working directory |
| `message` | User or assistant message |
| `toolCall` | LLM tool invocation |
| `toolResult` | Tool execution result |
| `compaction` | Context summarization |
| `branch_summary` | Branch point summary |

## Tree Structure

Sessions form a **tree** — not a linear list. This means:

```
User: "Build auth"
  Assistant: "I'll add JWT..."
    Tool: write auth.ts
    Tool: read auth.test.ts
  User: "Actually, use sessions instead"
    Assistant: "Ok, let me change..."
      Tool: edit auth.ts
    Tool: write session-auth.ts
```

You can **branch** at any point to try different approaches!

## Navigating with /resume

Type `/resume` in pi to see all your sessions:

```
┌─────────────────────────────────────────────┐
│  Sessions for ~/projects/my-app            │
├─────────────────────────────────────────────┤
│  ● main (current) - 47 messages            │
│  ○ auth-refactor - 23 messages             │
│  ○ feature-webhooks - 12 messages         │
└─────────────────────────────────────────────┘
```

Press Enter to continue the selected session.

<Callout variant="info" title="Session Switching">
You can have multiple sessions for the same project. Use `/resume` to switch between different work contexts.
</Callout>
