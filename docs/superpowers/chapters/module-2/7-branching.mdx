# Branching & forks

Sessions support branching — try different approaches without losing your main work.

## The /tree Command

Press `Esc` twice to open the session tree navigator:

```
┌─────────────────────────────────────────────┐
│  Session Tree                    Ctrl+O filter│
├─────────────────────────────────────────────┤
│  ○ User: Build auth                        │
│  ● Assistant: Using JWT                    │
│  ○ Tool: write auth.ts                     │
│  ○ Tool: read auth.test.ts                 │
│  ○ User: Actually use sessions             │
│  ● Assistant: Ok, switching approach       │
│  ○ Tool: edit auth.ts                      │
└─────────────────────────────────────────────┘
```

Navigate with arrow keys, jump to any point, or branch from there.

## Creating a Branch

### /fork

Create a new session from a previous user message:

```
/fork
```

This copies the path up to that point and opens an editor for your new prompt.

### /clone

Duplicate the current active branch into a new session:

```
/clone
```

Full history preserved, starts with an empty editor.

## Why Branch?

- **Compare approaches** — Try two different algorithms, see results side by side
- **Experiment safely** — Don't break working code
- **Consult old branches** — `/tree` to navigate back

<Callout variant="tip" title="Best Practice">
Create a branch when you're about to try something risky. If it works, merge (manually). If it fails, just navigate back to the main branch.
</Callout>
